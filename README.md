Project Showcase: Embedded Pulse Oximeter & Real-Time Biometric MonitorCourse Component: BMEN 2151: Prototyping for Biomedical Engineers   System Architecture: Embedded Systems / Biomedical Signal Processing / Hardware-in-the-Loop   Project OverviewThis project details the design, fabrication, and calibration of a noninvasive biomedical monitoring system capable of measuring real-time heart rate and blood oxygen saturation ($SpO_2$). The system features an integrated hardware-to-software pipeline that captures optical photoplethysmogram (PPG) waveforms, digitizes raw physiological data, and dynamically outputs metrics to a local user display.  Hardware Architecture & ToolingA closed-loop monitoring environment was fabricated using the following components and instrumentation:  MAX30102 Optical Biosensor: Utilizes dual-wavelength (Red and Infrared) LEDs and a photodetector to capture arterial pulsations.  Arduino Uno Microcontroller: Serves as the central processing unit, executing algorithmic logic, computing $I^2C$ sensor data, and managing system communication.  $I^2C$ OLED Display: Acts as the physical user interface to output real-time physiological metrics.  Precision Tools & Diagnostic Equipment: Soldering iron (alloy wire), heat guns with protective shrink-tubing for insulated wiring, digital multimeters for electrical continuity testing, and high-magnification dissecting scopes for micro-solder joint verification.  Engineering Methodology1. Hardware Fabrication & Quality ControlReliable micro-soldering is essential for stable medical device performance. Male headers were permanently attached to the MAX30102 sensor breakout board using an alloy solder iron.  Microscopic Inspection: Solder joints were evaluated under an AmScope dissecting microscope to verify structural integrity and identify defects.  Defect Identification & Rework: Microscopic analysis successfully detected a minor solder bridge on the upper communication pins. This unwanted bridge posed a short-circuit risk to the $I^2C$ data line and LED power paths, requiring precision thermal rework to guarantee long-term signal stability.  Insulation: Exposed copper interconnects were permanently sealed using mechanical heat-shrink tubing to prevent accidental shorts during operation.  2. Signal Processing & Mathematical ModellingThe core software architecture isolates human physiology by separating optical signals into two distinct vectors:  DC Component: The steady-state baseline representing light absorption from constant blood volume and static surrounding tissues.  AC Component: The dynamic, pulsatile waveform caused strictly by arterial expansion during cardiac cycles.  By isolating the AC fluctuations from the constant DC baseline, the device filters out background noise and targets user heart rate (derived via peak-to-peak beat intervals) and blood oxygenation.  Oxygen saturation is calculated using an approximate empirical model based on the "ratio of ratios" ($R$) formula:  $$R = \frac{\left(\frac{AC_{\text{red}}}{DC_{\text{red}}}\right)}{\left(\frac{AC_{\text{IR}}}{DC_{\text{IR}}}\right)}$$$$SpO_2 = 110 - 25R$$System Performance & VerificationDuring validation trials, the prototype successfully initialized, established steady state, and delivered continuous, low-latency biometric feedback.  Hemodynamic Tracking: The sensor captured real-time changes in finger blood volume—noting maximum absorption during ventricular systole and minimum absorption during diastole.
